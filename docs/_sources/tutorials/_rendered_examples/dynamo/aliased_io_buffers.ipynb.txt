        "\n\n# In-place aliased I/O: module-owned buffers\n\nThis is the PyTorch-canonical pattern for streaming inference: the cache\nlives inside the model via ``register_buffer``. The user simply calls\n``model(x)`` \u2014 no need to thread the cache through manually.\n\nHow it flows through the compile pipeline:\n\n* ``torch.export`` captures ``cache`` as a ``BUFFER`` input in the\n  ``graph_signature``; mutations to it become ``BUFFER_MUTATION``\n  output specs.\n* ``ExportedProgram.module()`` rewrites the buffer into a ``get_attr``\n  node plus a trailing ``aten.copy_(get_attr_buffer, slice_scatter_result)``\n  that represents the mutation.\n* Torch-TensorRT's ``lift_mutated_buffers`` pre-compile pass detects that\n  trailing ``copy_``, lifts the ``get_attr`` to a ``placeholder``, and\n  rebuilds the GraphModule so the engine treats the buffer as a regular\n  input binding.\n* The slice-scatter converter sees the cache as a network input and emits\n  ``IKVCacheUpdateLayer`` with aliased I/O.\n* The compiled result's lifted-buffer placeholders are rewritten in place\n  to ``get_attr`` reads from registered ``nn.Module`` buffers (via\n  ``inline_lifted_buffers_into_gm``). The buffer state lives on the\n  compiled module itself; ``forward`` takes only user inputs. Because the\n  result is a plain ``fx.GraphModule`` with buffers, it serializes through\n  ``torch_tensorrt.save`` / ``torch.export`` without any external wrapper.\n\nThe net effect: the engine writes through the buffer's storage in place,\nand the next call sees the updated state. No copy-back, no allocation\nper call.\n"

        "class StreamingKV(torch.nn.Module):\n    def __init__(self, b=1, h=4, s_max=16, d=8):\n        super().__init__()\n        self.register_buffer(\"cache_k\", torch.zeros(b, h, s_max, d))\n        self.register_buffer(\"cache_v\", torch.zeros(b, h, s_max, d))\n\n    def forward(self, x_k, x_v):\n        self.cache_k[:, :, 3:4, :] = x_k\n        self.cache_v[:, :, 3:4, :] = x_v\n        return self.cache_k.sum() + self.cache_v.sum()"
