        "\n\n# In-place aliased I/O: caller-owned tensors\n\nThis example shows the simplest in-place pattern: the caller owns a buffer\n(e.g. a KV cache), passes it into the compiled module on every call, and\nthe TensorRT engine mutates it in place. No fresh allocation per call, no\npost-engine copy.\n\nMechanically:\n\n* The model writes into a slice of one of its inputs:\n  ``cache[:, :, t:t+1, :] = update``.\n* The converter recognizes the ``slice_scatter`` pattern and emits a\n  ``IKVCacheUpdateLayer`` whose output is aliased to the cache input.\n* The C++ runtime binds the aliased output to the input's ``data_ptr``,\n  skipping ``at::empty``. The engine writes through that pointer directly\n  into the caller's tensor storage.\n* The user-facing return tuple contains only the model's explicit outputs\n  \u2014 the aliased \"mutation output\" is invisible to the caller.\n\nConstraints (from the TensorRT operator):\n\n* Cache shape must be 4-D ``[batch, heads, max_seq, head_dim]`` and fully\n  static.\n* Write dimension must be ``2`` (the sequence axis).\n* ``write_start + update_len <= max_seq``.\n\nIf the constraints aren't met the converter silently falls back to a\nregular scatter and aliasing isn't established (correctness is still\npreserved, but no in-place benefit).\n"
