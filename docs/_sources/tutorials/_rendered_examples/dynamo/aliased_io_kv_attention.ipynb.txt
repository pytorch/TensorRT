        "\n\n# Streaming attention with a static KV cache\n\nA realistic single-layer transformer attention block with a static-shape\nKV cache held as module buffers. This is the canonical PyTorch pattern\nfor streaming decoder inference: at each step the module takes a single\ntoken's hidden state, projects K and V, writes them into a fixed-size\ncache, and attends over the cache.\n\nCompared to the simpler aliased-I/O examples this one exercises:\n\n* ``LayerNorm``, ``Linear`` projections, ``scaled_dot_product_attention``\n* multi-head reshapes / transposes around the cache writes\n* the ``register_buffer`` + slice-write pattern for both K and V\n\nThe compiled engine emits two ``IKVCacheUpdateLayer`` ops (one each for\nK and V) with aliased outputs. The C++ runtime writes the new K/V\ndirectly into the buffer storage; the next step's attention reads the\nupdated cache without any copy.\n"

        "## Single-layer attention block\nThe model takes one timestep at a time and uses a compile-time\nconstant ``write_pos`` for the cache slot. In a real generation loop\nyou'd vary ``write_pos`` per step; a few practical recipes:\n\n* Recompile once per ``write_pos`` (cheap with engine caching).\n* Bake ``write_pos`` into the model's state via a buffer and increment\n  it inside ``forward`` (requires an extra in-place op pattern we\n  support separately).\n* Use the lower-level ``user_inputs`` flow and pass ``write_pos`` as\n  an integer argument to a wrapper that selects between pre-compiled\n  engines.\n\n"
      ]

        "class StaticKVAttention(torch.nn.Module):\n    def __init__(self, batch=1, max_seq=64, n_heads=4, head_dim=16, write_pos=3):\n        super().__init__()\n        self.batch = batch\n        self.n_heads = n_heads\n        self.head_dim = head_dim\n        self.hidden = n_heads * head_dim\n        self.write_pos = write_pos\n\n        # Static KV cache \u2014 fixed shape across the whole generation.\n        self.register_buffer(\"cache_k\", torch.zeros(batch, n_heads, max_seq, head_dim))\n        self.register_buffer(\"cache_v\", torch.zeros(batch, n_heads, max_seq, head_dim))\n\n        self.norm = torch.nn.LayerNorm(self.hidden)\n        self.q_proj = torch.nn.Linear(self.hidden, self.hidden, bias=False)\n        self.k_proj = torch.nn.Linear(self.hidden, self.hidden, bias=False)\n        self.v_proj = torch.nn.Linear(self.hidden, self.hidden, bias=False)\n        self.o_proj = torch.nn.Linear(self.hidden, self.hidden, bias=False)\n\n    def forward(self, hidden_states):\n        # hidden_states: [B, 1, H]\n        B = hidden_states.shape[0]\n        h = self.norm(hidden_states)\n\n        q = self.q_proj(h).view(B, 1, self.n_heads, self.head_dim).transpose(1, 2)\n        k = self.k_proj(h).view(B, 1, self.n_heads, self.head_dim).transpose(1, 2)\n        v = self.v_proj(h).view(B, 1, self.n_heads, self.head_dim).transpose(1, 2)\n\n        # In-place writes into the static KV cache. These two lines are\n        # what trigger the KV-cache aliasing fast path.\n        self.cache_k[:, :, self.write_pos : self.write_pos + 1, :] = k\n        self.cache_v[:, :, self.write_pos : self.write_pos + 1, :] = v\n\n        attn_out = F.scaled_dot_product_attention(q, self.cache_k, self.cache_v)\n        attn_out = attn_out.transpose(1, 2).contiguous().view(B, 1, self.hidden)\n        return self.o_proj(attn_out)"
